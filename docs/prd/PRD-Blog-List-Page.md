# PRD: 博客列表页面优化 (图文并排版)

## 1. 业务背景
为了提升用户浏览博客内容的体验，增强页面的视觉吸引力和信息传递效率，现计划对博客列表页进行重构。新的设计将采用经典的“左图右文”布局，使读者能通过缩略图快速建立第一印象。

## 2. 核心需求
- **布局升级**: 列表项由纯文字或简单排列改为左侧展示封面图，右侧展示文章元数据。
- **自适应响应**: 在移动端和桌面端提供一致且优化的阅读体验。
- **信息分层**: 明确标题、摘要、分类、日期及阅读量等信息的层级关系。
- **交互反馈**: 列表项悬停效果及流畅的加载体验。

## 3. 页面设计方案

### 3.1 布局结构 (Layout Structure)
采用“混合网格 (Mixed Grid)”布局风格，提升视觉层次感：

- **首屏置顶 (Featured Post)**：
  - 列表第一篇文章采用横向大卡片展示。
  - 布局比例：左侧图片约占 55%，右侧内容约占 45%。
  - 圆角：32px 极大圆角，营造柔和、现代感。
  - 信息排布：顶部左右分布日期与标签，中部展示大标题与摘要，底部展示统计数据。

- **后续列表 (Grid List)**：
  - 从第二篇文章开始采用两列网格布局（移动端为单列）。
  - 布局形式：上方图片（16:10 比例），下方内容。
  - 交互：整体卡片式悬停动效。

### 3.2 视觉规范 (Visual Specifications)
- **卡片设计**：
  - 背景：白色 (#FFFFFF)，边框：浅灰 (#F1F5F9)。
  - 阴影：低强度柔和阴影，Hover 时增强。
  - 圆角：`rounded-[32px]`。
- **排版 (Typography)**：
  - 标题：`font-bold`，置顶篇 `text-3xl`，网格篇 `text-xl`。
  - 摘要：`text-slate-500`，行高 `leading-relaxed`。
- **动效 (Animations)**：
  - 图片：Hover 时 `scale-105` 平滑放大（700ms 持续时间）。
  - 文字：标题 Hover 时颜色切换至蓝色 (`text-blue-600`)。

### 3.3 分页与加载 (Pagination & Loading)
- **分页策略**：采用标准数字分页。
- **每页数量 (PageSize)**：默认为 5 篇（配合混合布局，首页展示 1 大 + 4 小）。
- **分页组件样式**：
  - 采用圆点/数字按钮形式。
  - 包含“上一页”、“下一页”图标按钮。
  - 当前页高亮显示（蓝色背景）。
- **加载状态**：切换页面时显示 Skeleton 或 Loading 提示。
- **滚动行为**：切换分页后自动平滑滚动至页面顶部。

## 4. 技术规格

### 4.1 数据结构要求 (Frontend)
```typescript
interface BlogPost {
  id: string;
  title: string;
  excerpt: string;
  coverImage: string; // URL
  category: {
    id: string;
    name: string;
  };
  author: string;
  createdAt: string;
  viewCount: number;
}
```

### 4.2 UI 组件建议
- 使用 `aspect-ratio` 控制图片比例。
- 使用 `clamp()` 或 `line-clamp` 处理文本溢出。
- 引入 `Skeleton Screen` 作为数据加载时的占位符。

## 5. 验收标准
1. [ ] 页面在 1440px, 1024px, 768px, 375px 下均无布局错乱。
2. [ ] 无封面图的文章需显示统一的默认占位图。
3. [ ] 悬停动画平滑，无掉帧现象。
4. [ ] 标题和摘要的溢出省略逻辑生效。
5. [ ] 图片懒加载功能正常，减少首屏流量消耗。
