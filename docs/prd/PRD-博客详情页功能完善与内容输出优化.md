# PRD-博客详情页功能完善与内容输出优化

## 1. 文档概述
本文档旨在解决当前博客详情页内容显示缺失的问题，并对详情页的功能完整性进行系统性增强设计。

---

## 2. 问题诊断：详情页内容缺失
### 2.1 现象描述
用户点击博客列表进入详情页后，页面仅显示标题、标签等元数据，正文区域为空。

### 2.2 原因分析
经过对后端 `PublicController.getPostDetail` 及前端 `BlogDetailPage.tsx` 的代码审计，发现：
- **后端逻辑**: 后端已实现 Markdown 到 HTML 的转换逻辑（使用 `unified` 工具链），并输出了 `contentHtml` 字段。
- **爬虫存储**: 爬虫服务在导入文章时，将抓取到的内容存放在了 `contentMarkdown` 字段中。
- **潜在症结**: 
  1. **数据一致性**: 部分历史抓取的数据可能在 `contentMarkdown` 字段为空，或在转换过程中出现了异常。
  2. **前端渲染**: 前端通过 `dangerouslySetInnerHTML` 渲染 `post.contentHtml`，若该字段为空或 CSS 样式未加载，则无法显示。

---

## 3. 功能增强构思

为了提升博客的专业性与互动性，建议补全以下详情页功能：

### 3.1 内容展现类 (P0)
- **代码高亮优化**: 目前虽然引入了 `rehype-highlight`，但需要确保前端引入了对应的 CSS 主题（如 `github-dark` 或 `vscode-style`）。
- **图片预览 (Lightbox)**: 点击正文图片可放大查看原图，提升阅读体验。
- **文章更新时间**: 区分“发布时间”与“最后修改时间”，体现内容的时效性。

### 3.2 交互与闭环类 (P1)
- **点赞交互 (Real-time Like)**: 目前 `likes` 仅为展示，需要实现前端点击 -> 后端原子递增 -> 状态回显。
- **社交分享**: 生成文章分享海报或提供一键分享至微信、X (Twitter)、GitHub 的功能。
- **评论系统**: 接入第三方评论系统（如 Giscus 或 Waline），增加读者互动。

### 3.3 留存与导航类 (P2)
- **相关文章推荐**: 基于标签 (Tags) 或向量搜索 (Qdrant) 推荐相似内容。
- **阅读进度条**: 顶部显示吸顶的进度条，告知用户阅读进度。
- **文章打赏**: 提供微信/支付宝赞赏码展示。

---

## 4. 详细需求说明

### 4.1 [后端] 详情接口健壮性增强
- **需求**: 当 `contentHtml` 转换失败或为空时，返回友好的错误提示而非空字符串。
- **逻辑**: 
  ```typescript
  if (!post.contentMarkdown) {
    // 触发自愈逻辑或记录异常日志
  }
  ```

### 4.2 [前端] Markdown 样式补全
- **需求**: 确保 `src/styles/markdown.css` 包含基础的 Typography 样式（列表缩进、引用块背景、表格边框等）。
- **样式参考**: GitHub 风格或 Tailwind Typography。

### 4.3 [交互] 阅读量统计自增
- **需求**: 每次成功加载详情页时，后端 `stats.views` 应自动 +1。
- **实现**: 在 `getPostDetail` 接口内部异步触发数据库更新。

---

## 5. 验收标准 (Acceptance Criteria)
1. 详情页能够正确显示抓取自掘金的 Markdown 转换后的 HTML 内容。
2. 代码块拥有明显的语法高亮效果。
3. 页面右侧目录 (TOC) 点击可精准跳转至对应标题。
4. 详情页刷新后，阅读量 (Views) 数值增加。
