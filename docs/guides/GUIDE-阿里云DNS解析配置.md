# 阿里云 DNS 解析配置实操手册

本文档指导你如何在阿里云“云解析DNS”后台为你的个人博客项目配置前端和后端的访问地址。

---

## 1. 前端解析配置（指向 Vercel）

如果你打算把前端部署在 Vercel，请在阿里云后台点击 **“添加记录”**，并按以下内容填写：

| 设置项 | 填写内容 | 说明 |
| :--- | :--- | :--- |
| **记录类型** | `CNAME` | 将子域名指向另一个域名 |
| **主机记录** | `www` | 最终访问地址为 `www.你的域名.com` |
| **解析线路** | `默认` | 保持默认即可 |
| **记录值** | `cname.vercel-dns.com` | **这是 Vercel 的标准地址** |
| **TTL** | `10分钟` | 解析生效的缓存时间 |

> **提示**：如果你想让不带 `www` 的主域名（如 `你的域名.com`）也能访问，可以再加一条主机记录为 `@` 的 `CNAME` 记录，记录值同样指向 `cname.vercel-dns.com`。

---

## 2. 后端 API 解析配置（指向服务器）

根据你选择的后端托管方式，填写内容会有所不同：

### 方案 A：使用 Railway / Render（推荐）
| 设置项 | 填写内容 | 说明 |
| :--- | :--- | :--- |
| **记录类型** | `CNAME` | |
| **主机记录** | `api` | 最终接口地址为 `api.你的域名.com` |
| **记录值** | `你的项目名.railway.app` | **填写平台分配给你的服务域名** |

### 方案 B：使用自己的阿里云服务器 (ECS)
| 设置项 | 填写内容 | 说明 |
| :--- | :--- | :--- |
| **记录类型** | `A` | 将域名直接指向一个 IP 地址 |
| **主机记录** | `api` | |
| **记录值** | `47.xx.xx.xx` | **填写你 ECS 服务器的公网 IP** |

---

## 3. 配置后的验证步骤

配置完成后，由于 DNS 需要在全球生效（通常几分钟内），你可以通过以下方式验证：

1.  **终端验证**：
    打开终端（Terminal）执行：
    ```bash
    ping www.你的域名.com
    ping api.你的域名.com
    ```
    如果返回的 IP 地址不再是报错，或者指向了 Vercel/服务器的 IP，说明配置成功。

2.  **HTTPS 证书**：
    -   **Vercel**：当你添加完解析后，Vercel 会自动检测并为你生成免费的 SSL 证书。
    -   **后端**：如果你用的是 Railway/Render，它们也会自动生成证书。如果是自建服务器，你之后需要使用 Certbot 等工具申请。

---

## 4. 后续步骤

配置好 DNS 后，回到 Vercel 的项目设置页面，点击 **"Refresh"** 或 **"Check"**。一旦它显示 **"Valid Configuration"**，你的前端网站就已经正式在互联网上“安家”了！
