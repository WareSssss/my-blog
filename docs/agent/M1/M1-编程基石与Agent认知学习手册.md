## 1. 编程基石 (M1.1)

### 1.1 Python 高级特性
- **列表推导式 (List Comprehensions)**: 
  - 语法：`[expression for item in iterable if condition]`
  - 作用：简洁地创建列表。在 AI 处理数据集时极度常用。
- **装饰器 (Decorators)**:
  - 作用：在不改变原函数代码的情况下增加功能。
  - 应用：NestJS 的 `@Controller()`, `@Get()` 本质上就是装饰器思想。
- **异步编程 (Async/Await)**:
  - 核心：非阻塞 I/O。AI 接口响应通常较慢，必须掌握异步处理以防止系统假死。

### 1.2 TypeScript 核心 (针对本项目)
- **类型安全**: 掌握接口 (`interface`) 与类型 (`type`)，确保在调用 AI 返回的 JSON 时不会因为字段缺失崩溃。
- **依赖注入 (DI)**: 理解 NestJS 如何管理服务实例，这是构建大型 Agent 系统的架构基础。

## 2. 深度学习“黑盒”解密 (M1.2)

### 2.1 神经元与权重 (Weights)
- **输入与输出**: 神经网络通过输入（如文字的向量）乘以权重，再加上偏置，经过激活函数输出结果。
- **训练本质**: 训练模型的过程就是不断调整这些“权重”的过程，直到输出结果接近真实答案。

### 2.2 前向传播与反向传播
- **前向传播**: 数据从输入层流向输出层得到预测值的过程。
- **反向传播 (Backpropagation)**: 计算预测值与真实值的误差，并将误差反向传回网络，利用**梯度下降**更新权重。

## 3. Agent 架构模型认知 (M1.3)

### 3.1 大脑 (LLM)
- Agent 的核心推理引擎。它负责解析指令、规划步骤、生成回复。
- **限制**: 知识截止日期、缺乏实时工具、Token 窗口有限。

### 3.2 规划 (Planning)
- **思维链 (CoT)**: 引导模型“一步步思考”，提高复杂问题的解决成功率。
- **任务拆解**: 将大目标（如“写一篇博客”）拆解为（搜索、总结、配图、排版）。

### 3.3 记忆 (Memory)
- **短期记忆**: 对话上下文。
- **长期记忆**: 向量数据库（本项目中的 Qdrant）。通过 Embedding 技术将知识存入，按需检索。

### 3.4 工具 (Tools)
- Agent 的“手脚”。通过 **Function Calling**，LLM 可以主动提出调用外部 API（如本项目中的爬虫脚本、数据库读写）。
