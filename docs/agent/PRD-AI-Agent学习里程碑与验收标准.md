# PRD-AI Agent 学习里程碑与验收标准

## 1. 文档概述
本文档将《AI Agent 学习全景路线图》转化为可落地的项目里程碑，并为每个阶段定义明确的“交付物”与“验收标准”，确保学习过程可量化、可验证。

---

## 2. 学习里程碑规划

### 🚩 Milestone 1: 编程基石与 Agent 认知 (基础期)
**目标**: 能够读懂 AI 项目源码，理解 Agent 的基本运转逻辑。
- **M1.1**: 掌握 Python 核心语法（推导式、装饰器）与 TypeScript 异步编程。
- **M1.2**: 理解深度学习“黑盒”原理（权重、前向传播、损失函数）。
- **M1.3**: 建立 Agent 架构模型（大模型 + 规划 + 记忆 + 工具）。

### 🚩 Milestone 2: RAG 工程化与系统设计 (进阶期)
**目标**: 具备构建生产级知识库检索系统与复杂逻辑处理能力。
- **M2.1**: 掌握 RAG 全链路（分块、向量化、检索、重排）。
- **M2.2**: 深入 NestJS 架构，实现复杂的业务逻辑与 AI 接口集成。
- **M2.3**: 掌握向量数据库（Qdrant）的 CRUD 与索引优化。

### 🚩 Milestone 3: 架构深度与性能优化 (专家期)
**目标**: 理解模型底层机制，具备系统调优与安全防护能力。
- **M3.1**: 攻克 Transformer/Attention 机制，理解 Token 生成逻辑。
- **M3.2**: 实现 Token 消耗治理与安全注入防护方案。
- **M3.3**: 具备独立设计多 Agent 协作系统 (MAS) 的能力。

---

## 3. 详细验收标准 (Acceptance Criteria)

### 3.1 [基础阶段] 验收标准
- [ ] **代码阅读**: 能够无障碍阅读本项目中 `crawler.service.ts` 的异步流转逻辑。
- [ ] **黑盒推演**: 能够向他人解释：当用户输入一段话，LLM 是如何通过 Embedding 变成数字并找到知识库内容的。
- [ ] **环境搭建**: 能够独立完成本项目在本地的部署、数据库迁移与 API 联调。

### 3.2 [进阶阶段] 验收标准
- [ ] **RAG 调优**: 能够通过修改 `chunkSize` 或检索 `topK` 参数，显著提升知识库回复的准确度。
- [ ] **工具扩展**: 能够为本项目新增一个 Agent 工具（例如：查询天气或发送邮件通知）。
- [ ] **数据库切换**: 能够解释为什么在本项目中使用 Qdrant 而非传统的模糊查询，并能对比其性能差异。

### 3.3 [专家阶段] 验收标准
- [ ] **原理解构**: 能够手绘 Transformer 的 Encoder/Decoder 简易架构图并说明 Attention 的作用。
- [ ] **成本控制**: 能够设计一套逻辑，对长对话进行摘要压缩，以减少 30% 以上的 Token 消耗。
- [ ] **漏洞排查**: 能够识别并修复常见的 Prompt 注入漏洞，确保系统不会被恶意指令操控。

---

## 4. 持续评估机制
- **每周复盘**: 每周对照里程碑检查进度，未达标项需记录原因并调整学习优先级。
- **实战交付**: 每个 Milestone 结束后，必须在本项目中提交至少一个相关的 Feature 代码（如：M2 结束后，提交一个优化后的搜索功能）。
