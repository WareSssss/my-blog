# AI Agent 学习全景路线图 (Learning Path)

根据您提供的学习路线图，结合本项目（基于 NestJS + Qdrant + RTP 协议）的实战背景，为您整理了一份从基础到进阶的学习指南。

---

## 🟢 第一阶段：夯实基础 (Foundation)

### 1.1 编程语言基石
- **Python 基础**: 虽然本项目后端使用 TypeScript，但 AI 领域的顶层设计、数据处理和模型微调（Fine-tuning）几乎全是 Python。
  - **重点**: 列表推导式、标准库函数、装饰器。
- **TypeScript/Node.js**: 理解异步编程（Async/Await）、装饰器（NestJS 核心）以及类型系统。

### 1.2 深度学习常识
- **核心概念**: 理解神经网络的输入层、输出层、神经元。
- **运行机制**: 了解梯度下降、反向传播、权重更新的基本逻辑。不需要手写公式，但要理解“模型是如何学习的”。

### 1.3 Agent 核心概念
- **三位一体**: 深刻理解 **记忆 (Memory)**、**工具 (Tools)**、**知识库 (Knowledge Base)** 如何协同工作。
- **本项目对应**:
  - 记忆 -> `ChatSession` (Postgres)
  - 工具 -> `CrawlerService`, `PrismaService`
  - 知识库 -> `Qdrant` (Vector DB)

---

## 🟡 第二阶段：工程进阶 (Intermediate)

### 2.1 架构思想 (Harness Thinking)
- **Agent 系统设计**: 学习如何构建一个健壮的智能体系统。
- **上下文管理**: 学习上下文压缩策略、滑动窗口记忆等，解决 LLM Token 限制问题。

### 2.2 RAG 深度实战
- **分词与索引**: 了解不同 Chunking（分块）策略对检索质量的影响。
- **检索策略**: 学习向量检索原理、重排 (Re-ranking) 技术。
- **数据库对比**: 横向对比 **Qdrant** (本项目使用)、Milvus、Neo4j (图数据库) 的优劣。

### 2.3 异步与接口
- **Web 框架**: 深入学习 NestJS (本项目) 或 FastAPI (AI 常用)。
- **函数调用**: 掌握 LLM 如何根据描述选择并调用外部函数 (Function Calling)。

---

## 🔴 第三阶段：深度探索 (Advanced)

### 3.1 Transformer 架构
- **核心机制**: 深入研究 **Self-Attention (自注意力)** 机制的原理。
- **源码走读**: 尝试阅读基础的 Transformer 实现代码。

### 3.2 性能与安全边界
- **Token 治理**: 学习如何降低推理成本，防止 Token 浪费。
- **系统安全**: 学习提示词注入防护、敏感内容过滤。

---

## 💡 资深建议 (Pro Tips)

1. **掌握官方文档**: 所有的二手资料（包括本指南）都没有官方文档（如 OpenAI, LangChain, NestJS 官网）权威。
2. **关注前沿资讯**: 推荐关注“腾讯云开发者”等技术公众号，保持对大模型（LLM）迭代的敏感度。
3. **拒绝“喂饭式”学习**: AI 领域变化极快，必须保持思考与挑战，多去折腾复杂的 Bug（正如我们之前处理数据库迁移 P3008/P3017 错误的过程）。

---
**学习结语**: 所有的理论最终都要落实到代码上。建议您在学习这些概念时，同步在本项目中尝试修改一个功能（例如：将 Qdrant 的检索逻辑升级为“混合检索”）。
